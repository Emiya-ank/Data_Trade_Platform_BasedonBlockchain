## 🚀 快速开始

### 1. 基本使用流程

```go
// 步骤 1：定义你的谓词
circuit := &RangePredicateCircuitFull{
    Plaintext: plaintext,
    TextLen:   textLen,
    CommitX:   commitment,      // 所有谓词的共同锚点
    RangeMin:  big.NewInt(100),
    RangeMax:  big.NewInt(200),
    TargetIdx: big.NewInt(0),
}

// 步骤 2：编译、Setup、Prove、Verify（见 test_predicates.go）
r1cs, _ := frontend.Compile(ecc.BN254.ScalarField(), r1cs.NewBuilder, circuit)
pk, vk, _ := groth16.Setup(r1cs)
proof, _ := groth16.Prove(r1cs, pk, fullWitness)
groth16.Verify(proof, vk, publicWitness)
```

```go
manager := NewPredicateManager()

// 查看所有谓词

    fmt.Println(name)  // range_check, hash_check, disclosure

```

## 🔗 关键概念

### CommitX 的作用

所有谓词都以 **`CommitX`** 作为**公开链接点**：

```
CommitX = Pedersen(x, r)  ← 数据承诺（公开）
  ↓
  ├─ 私有 x 用于 φ₁（范围谓词）
  ├─ 私有 x 用于 φ₂（披露谓词）
  └─ 私有 x 用于 φ₃（哈希谓词）

验证方只需验证 CommitX + π₁ + π₂ + π₃
即可确认数据 x 满足所有谓词
```

## 📊 约束分解

每个谓词的约束构成：

### 范围谓词
```
约束 1：索引有效性        (10 约束)
约束 2：下界检查 (ToBinary) (70 约束)
约束 3：上界检查 (ToBinary) (70 约束)
总计：~150 约束
```

### 选择性披露谓词
```
约束 1：掩码布尔性         (1024 约束，每个元素 1 个)
约束 2：掩码连续性         (1024 约束，连续性检查)
约束 3：披露一致性         (1024 约束，x' = x * b)
总计：~3000+ 约束
```

### 哈希一致性谓词
```
Poseidon2 哈希约束：
  - 轮函数展开：8 轮 × 8 状态 × 多个约束
  - ~200 约束
总计：~200 约束
```

## 🔄 工作流程

```
┌─────────────────────────────────────────┐
│  [离线] Seller 侧                        │
│                                          │
│  x (明文) → 计算 CommitX、加密、验算    │
│  设定各谓词的公开参数                    │
└────────────┬────────────────────────────┘
             │ 公开：CommitX、谓词参数
             ▼
┌──────────────────────────────────────────┐
│  [生成] Groth16 Setup 和 Prove            │
│                                           │
│  φ₁ 电路 → Setup → pk₁, vk₁ → Proof π₁  │
│  φ₂ 电路 → Setup → pk₂, vk₂ → Proof π₂  │
│  φ₃ 电路 → Setup → pk₃, vk₃ → Proof π₃  │
└────────────┬───────────────────────────┘
             │ 发送：π₁, π₂, π₃ + CommitX
             ▼
┌──────────────────────────────────────────┐
│  [验证] Buyer 侧                          │
│                                           │
│  Verify(π₁, vk₁, CommitX) ? ✓           │
│  Verify(π₂, vk₂, CommitX) ? ✓           │
│  Verify(π₃, vk₃, CommitX) ? ✓           │
│                                           │
│  所有验证通过 → 数据满足所有谓词! ✓      │
└──────────────────────────────────────────┘
```

## ⚠️ 常见问题

### Q: 为什么所有谓词都需要 CommitX？
**A**: CommitX 是数据的公开承诺。不同谓词验证同一数据的不同属性，但都指向同一个 CommitX，确保数据一致性。

### Q: 能否只验证一个谓词？
**A**: 完全可以。谓词之间独立，可以独立证明和验证。组合多个谓词是为了表达更复杂的条件。

### Q: 如何添加新谓词？
**A**: 见上面"扩展"章节，实现 `PredicateCircuit` 接口即可。

### Q: CommitX 会暴露数据吗？
**A**: 不会。CommitX 是单向承诺（Pedersen），无法从 CommitX 恢复原数据。
